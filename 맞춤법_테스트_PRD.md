# 맞춤법 레벨 테스트 (가칭: 내 맞춤법 티어는?) PRD

## 1. 프로젝트 개요 (Project Overview)
- **목적**: 현대인들의 맞춤법 파괴 현상을 재미있게 꼬집고, 올바른 맞춤법을 알리기 위한 바이럴 웹 서비스
- **목표**: 10만 명 이상의 유저 참여 및 SNS 자발적 공유 (바이럴 루프 형성)
- **형태**: 모바일 최적화된 웹 기반 객관식 퀴즈 게임 및 결과 공유 페이지
- **배포 방식**: Cloudflare Pages를 이용한 정적 파일 배포 (빠른 로딩 및 대규모 트래픽 대비)

## 2. 타겟 오디언스 (Target Audience)
- 10대~30대 SNS(인스타그램, 카카오톡) 적극 이용자
- 친구들과 가벼운 심리테스트/퀴즈 결과를 공유하고 비교하기 좋아하는 사람
- 평소 맞춤법에 민감하거나, 반대로 맞춤법을 자주 틀려 지적받는 사람

## 3. 코어 기능 (Core Features)

### [3.1] 랜딩 페이지 (시작 화면)
- **훅(Hook)이 있는 타이틀**: "한국인 90%가 틀리는 맞춤법, 당신의 티어는?"
- **시작하기 버튼**: 클릭 시 바로 퀴즈 시작 (로그인/회원가입 등 이탈 요소 완벽히 제거)
- **동인 요소**: 현재까지 테스트에 참여한 총 유저 수 표시 (신뢰도 및 호기심 상승 효과)

### [3.2] 퀴즈 플레이 화면
- **문항 수**: 10~12개 문항 (빠르게 이탈하지 않도록 1~2분 내외로 소요되게 설계)
- **퀴즈 형태**: 두 개의 단어 카드 중 올바른 맞춤법을 제한 시간 내에 선택 (예: '어의없다' vs '어이없다', '되' vs '돼')
- **UI/UX**: 프로그레스 바(진행도) 표시, 빠르고 직관적인 터치 반응 (게임 같은 타격감 및 효과음 적용 고려)

### [3.3] 결과 화면 (바이럴의 핵심 ★)
- **결과 산출**: 맞힌 개수에 따른 '레벨/티어' 부여 
  - *예시: 세종대왕(12개), 국어문법학자(10~11개), 일반인(7~9개), 0개국어(4~6개), 침팬지(0~3개)*
- **뼈 때리는 멘트**: 각 레벨에 맞는 유머러스하고 자극적인 상세 멘트로 공유 욕구 자극
- **오답 노트**: 내가 틀린 문제와 올바른 맞춤법을 리스트 형태로 제공
- **공유 액션 버튼 (CTA)**:
  1. **카카오톡 공유하기**: 템플릿을 활용해 내 등급과 함께 "친구에게 도전장 내기" 유도
  2. **인스타그램 공유**: 스토리 규격(9:16)에 맞는 결과 이미지 다운로드 또는 캡처 가이드 (html2canvas 등 활용)
  3. **링크 복사**: 클립보드 복사 토스트 팝업

## 4. 기술 스택 및 구조 (Tech Stack & Architecture)
- **Frontend**: 웹 표준 HTML/CSS/Vanilla JS 또는 React (Vite) / Next.js의 Static Export 활용
- **Data (상태 관리)**: 별도의 Backend API나 DB 없이 `questions.json` 같은 정적 파일이나 코드 내 배열로 문제 관리 (비용 제로, 속도 최적화)
- **Hosting**: Cloudflare Pages (글로벌 CDN 캐싱으로 10만 명 이상의 트래픽이 몰려도 서버 다운 없이 무료/저비용으로 수용 가능)
- **Analytics**: Google Analytics 4 (GA4) - 사용자 수, 이탈률, "공유하기" 버튼 클릭 이벤트 트래킹
- **API**: 카카오톡 공유 API (Kakao JS SDK) 적용

## 5. 바이럴 루프 설계 (Viral Loop Strategy)
- **결과 뷰의 직관성**: 인스타 스토리나 카카오톡에 올라왔을 때, 다른 사람이 보고 "나도 해봐야지"라는 생각이 즉각적으로 들게 디자인
- **오픈그래프(OG) 메타 태그 최적화**: 카카오톡/슬랙 등에 링크를 붙여넣었을 때 나오는 썸네일 이미지와 제목, 설명을 매력적으로 세팅 (예: "제 맞춤법 티어는 [침팬지]입니다. 당신의 티어는?")
- **승부욕 자극**: 평균 점수 대비 내 점수의 위치(상위 X%)를 알려주어 지인들 간의 비교 심리 자극

## 6. MVP 마일스톤 (추천 개발 순서)
1. **문제 기획**: 사람들이 가장 헷갈려 하는 맞춤법 데이터 수집 (12문제 세트 구성)
2. **UI/UX 디자인 기획**: 모바일 퍼스트 기준, 텍스트 가독성이 높고 버튼이 큼직한 화면 설계
3. **프론트엔드 퍼블리싱 및 로직 구현**: 화면 이동, 점수 계산 로직, 결과 매핑 기능 구현
4. **공유하기 시스템 부착**: HTML2Canvas를 이용한 이미지 추출, 카카오 SDK 연동, OG 태그 작업
5. **정적 빌드 및 배포**: Cloudflare Pages 연결 (GitHub 연동을 통한 CI/CD 설정)
6. **시드 풀기**: 개발 완료 후 지인 단톡방, SNS 커뮤니티(에타 등)에 1차 배포하여 바이럴 시작
